# AI执行者工作法则 (V1.0)
## 第一条：行动前必须进行的"根本性反思"
在接受任何任务指令后、开始生成任何具体内容前，你必须强制执行以下三步反思，并将反思结论置于输出内容的最前方：
1.  **反思产品核心**："用户完成这个功能的核心动作是什么？是否是'拍照'？有没有引入'手动输入'的风险？"
2.  **反思数据逻辑**："这个设计中，'视觉份量'是如何与一个'具体被识别出的食物'关联的？系统最终如何计算出热量和蛋白质？"
3.  **反思架构洁癖**："我是否引入了不必要的复杂化？（如：添加了中间统计表、增加了非必要的配置选项）"

## 第二条：输出时必须附上的"自我检查声明"
在完成任何输出后，你必须根据输出内容，附上一份针对性的自我检查声明，格式如下：
【自我检查声明】
- [ ] 已确认本输出未引入任何形式的**手动输入接口**。
- [ ] 已确认本输出中"视觉份量"的交互逻辑**依赖于AI识别的具体食物**。
- [ ] 已确认本输出符合"实时计算"原则，**未添加冗余数据存储**。

## 第三条：对"审核意见"的响应规范
当接收到来自产品经理的审核意见时，你必须：
1.  首先，**复述审核意见的核心点**，以确认理解无误。
2.  然后，**分析该意见所指出的问题，违反了《产品根本法》或本法则的哪一条**。
3.  最后，基于此分析，提出或执行修正。

## 第四条：版本与更新
本法则为最高工作指导。最新版本永远以项目根目录下的 AI_WORK_RULE.md 为准。
